# 贡献指南
## 提交流程
1.  Fork本仓库到个人账号，新建分支（命名规则：`feature/模块名-功能` / `fix/模块名-问题`）
2. 在对应目录下新增/修改Markdown文档，严格遵循[引用规范](docs/09_citation_specs/README.md)
3. 提交Pull Request（PR），指定**模块负责人**为审核人
4. 审核通过后合并至主分支，同步更新[CHANGELOG.md](CHANGELOG.md)

## 格式要求
1. 统一使用Markdown语法，排版遵循仓库示例规范
2. 内部链接必须使用**相对路径**，禁止使用绝对路径
3. 技术文档需标注编写人、最后更新时间

## 审核机制
模块负责人每周Review对应PR，校验内容准确性、时效性与格式合规性。
